# wiki-query — 問い合わせ

Wiki（`wiki/`）に蓄積した知識を根拠に、引用付きで質問に答える。RAGのように毎回ゼロから探すのではなく、**既にある統合知識へ素早く到達する**。

**開始前に `wiki/_schema.md`（規約）を確認すること。** 還元時のページ作成は `wiki/_templates/` の雛形を使う。

## 検索: qmd を使う

第一の検索手段は **qmd**（`/qmd-setup` で導入するローカル検索）。

```bash
qmd query "<質問のキーワード>" -c wiki --json -n 8   # ハイブリッド+リランク（最良）
```

- ヒットしたページを `qmd get "<path>"` で取得して読む。
- `raw` コレクションも登録済みなら、生ソースまで遡りたいとき `-c raw` で検索する。
- **qmd が未インストール/未セットアップの場合**: `qmd --version` で確認し、未導入なら `/qmd-setup` を案内する。導入できないときは **フォールバックとして `wiki/index.md` を入口に** 関連ページを辿り、必要なら `grep` でWiki全体を検索する。
- qmd を MCP サーバとして導入済みなら、CLIの代わりに qmd の `query`/`get` ツールを使ってよい。

## 手順

1. **検索する** — 上記 qmd（またはフォールバックの index/grep）で質問に関連するページを特定する。
2. **掘る** — 関連ページ（entities / concepts / decisions / releases / backlog / sources / overview）を読む。リンクを辿って関連を広げる。
   - 根拠が見つからなければ「Wikiに情報が無い」と正直に言い、`/wiki-ingest` で取り込むべきソースや Web 検索を提案する。
3. **統合して答える** — 出所を `[[ファイル名]]`（と必要なら生ソース）で明示。複数ページにまたがる論点は統合し、矛盾があれば指摘する。
   - 形式は内容に合わせる: 文章 / 比較表 / mermaid 図 / ADRドラフト / Marp スライドなど。
4. **Wikiへ還元する（重要）** — 回答が再利用価値を持つなら、人間に確認のうえ新しいWikiページ（多くは `concepts/` か `decisions/`、雛形は `wiki/_templates/`）として保存し、index と関連ページにリンクを張る。還元したら `qmd embed` で検索インデックスも更新する。
5. **log に追記** — 重要なクエリや還元を行った場合:
   ```
   ## [YYYY-MM-DD] query | <問いの要約>
   - 参照: [[...]]
   - 還元したページ: [[...]]（あれば）
   ```

## 原則
- 出所のない断定をしない。Wikiに無いことは推測せず「未確認」と伝える。
- 良い回答をチャット履歴に消えさせない。還元を常に検討する。
