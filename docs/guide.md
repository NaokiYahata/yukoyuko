# テンプレート利用ガイド（LLM Project Base）

このリポジトリは「**複利的に育つWiki**」を中心に、**月次など短いサイクルで「決まったものから順にリリースする」イテレーティブな進行をAI（Claude Code）と行う** ためのテンプレートです。滝モデルのように最初に全工程を決めることはしません。

質問のたびにゼロから生ソースを探すRAGではなく、AIが知識を一度コンパイルして恒久的に最新化し続けます。相互リンク・矛盾の指摘・統合済みの考察が常に蓄積され、ソースを足すほど・問うほどWikiが賢くなります。

> 人間の仕事 = ソース収集・方向づけ・良い問い・意思決定。AIの仕事 = それ以外（要約・相互参照・整理・記帳・矛盾検出）。

## 使い方

1. このリポジトリを **「Use this template」** から新しいプロジェクトリポジトリとして作成。
2. クローンして、そのディレクトリで Claude Code を起動。
3. **`/project-init`** を実行してプロジェクトの目的・ゴール・ステークホルダーを設定（README もプロジェクト固有の内容へ書き換わる）。
4. **`/qmd-setup`**（任意・推奨）で検索エンジン qmd をインストール・セットアップ。
5. `raw/_originals/` に資料（pptx/pdf/xlsx/docx など）を置く。テキスト/mdならそのまま `raw/` へ。
6. **`/wiki-ingest`** で取り込み（バイナリは markdown に抽出して `raw/` 保存）、`wiki/` が育ち始める。
7. 以降、ソースを足しては取り込み、`/wiki-query` で問い、定期的に `/wiki-lint`。

Obsidian で `wiki/` フォルダを開くと、グラフビューで知識の形を見ながらブラウズできます（Obsidian = ビューア、Claude = 書き手、Wiki = 成果物）。エディタで `wiki/` を直接編集したときは **`/wiki-sync`** を実行すると、その差分を検出してリンク張り直し・index 同期・他ページとの矛盾チェックまで波及させます。

## 構成（3レイヤー）

| レイヤー | 場所 | 所有者 | 説明 |
|---|---|---|---|
| Raw（生ソース） | `raw/` | 人間 | 元バイナリ（pptx/pdf/xlsx/docx）は**Git管理せず**、抽出した markdown のみコミット。AIは読むだけ。 |
| Wiki（知識ベース） | `wiki/` | AI | AIが生成・維持する相互リンクされたmarkdown群。 |
| Schema（スキーマ） | `wiki/_schema.md` | 人間+AI | Wikiの構造・規約・ワークフロー定義。Wikiと一緒に移動する独立ファイル。 |

`CLAUDE.md` は薄く保ち、スキル早見表とプロジェクト固有設定・コード規約を置きます（スキーマは持ちません）。

```
raw/
├── _originals/   元バイナリ（git管理外）   wiki/
├── <slug>.md     抽出済みテキスト          ├── _schema.md  Wikiの仕様（規約）
└── assets/       画像など                  ├── _templates/ ページ雛形
                                            ├── index.md    全ページカタログ
                                            ├── log.md      時系列ログ
                                            ├── overview.md 全体像/thesis/現在のリリース
                                            ├── backlog.md  未割当の要求・アイデア
                                            ├── releases/   リリース(月)ハブ YYYY-MM.md
                                            ├── sources/    ソース要約
                                            ├── entities/   人物/組織/システム
                                            ├── concepts/   概念/論点
                                            └── decisions/  意思決定記録(ADR)
```

## スキル（`/<name>` で起動）

| スキル | 用途 |
|---|---|
| `/project-init` | 新規プロジェクトの初期化 |
| `/qmd-setup` | 検索エンジン qmd のインストール・セットアップ（初回のみ） |
| `/wiki-ingest` | `raw/` のソース（pptx/pdf/xlsx/docx は抽出）をWikiへ取り込み・波及更新 |
| `/wiki-sync` | エディタでの `wiki/` 直接編集や実装コードの変更を検出し、リンク・index・矛盾＋コードとの整合（ドリフト）を波及更新 |
| `/wiki-query` | qmdでWikiを検索し、引用付きで回答・還元 |
| `/wiki-lint` | 矛盾・陳腐化・孤立ページ等の健全性チェック |

## 検索（qmd）

Wikiの検索には [qmd](https://github.com/tobi/qmd)（ローカルのBM25/ベクトル/ハイブリッド検索）を使います。`/qmd-setup` でインストール〜インデックス作成まで行います（Node.js≥22、初回はモデル約2GBを自動DL）。導入しなくても index.md ベースで動作します。

新規ソースを取り込んだら `qmd update`（再インデックス）→ `qmd embed` の順で検索インデックスを更新します（`qmd embed` だけでは新規ファイルが検索に入りません）。

## リリースモデル（イテレーティブ）

最初に全工程を決める滝モデルではなく、短いサイクルで回します。

```
backlog に貯める → 直近リリースだけ詳細化（releases/YYYY-MM.md を作りスコープ確定）
→ 関連ページに release: YYYY-MM タグを付けて作業 → リリース
→ ハブに振り返りを追記し status: released → 未完は backlog/次リリースへ持ち越し
```

- **`backlog.md`**: 未割当の要求・アイデアの受け皿（最初に詳細化しない、1行で貯める）
- **`releases/YYYY-MM.md`**: リリース月ごとのハブ（ゴール/スコープ/振り返り/持ち越し）
- **`release: YYYY-MM` タグ**: 任意のページ（decisions/concepts/sources等）を特定リリースに紐づけ

Wikiの規約・ワークフローの詳細は [`wiki/_schema.md`](../wiki/_schema.md) を参照してください。

このテンプレートは **LLM Wiki パターン**（生ソースを毎回再検索するRAGではなく、AIが知識を恒久的に整理・最新化し続ける知識ベース運用）の考え方に基づきます。ドメインに合わせて `wiki/_schema.md` とスキルを自由に共進化させてください。
