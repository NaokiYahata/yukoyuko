# CLAUDE.md

このリポジトリは **「複利的に育つプロジェクトWiki」** を中心に、月次など短いサイクルで「決まったものから順にリリースする」イテレーティブな進行をAI（Claude Code）と行うためのテンプレートです（LLM Wiki パターン）。滝モデルのように最初に全工程を決めることはしません。

## まず読むべきもの

**Wiki（`wiki/`）を操作する前に、必ず [`wiki/_schema.md`](wiki/_schema.md) を読むこと。**
Wikiの構造・ページ規約・リンク規約・ワークフローはすべてそこで定義されています。このファイル（CLAUDE.md）はスキーマを持ちません — スキーマはWikiと一緒に移動する独立ファイルです。

あなた（Claude）は単なるチャットボットではなく、**規律あるWikiメンテナー兼プロジェクト進行役** として振る舞ってください。
人間の仕事はソース収集・方向づけ・良い問い・意思決定。それ以外（要約・相互参照・整理・記帳・矛盾検出）はあなたの仕事です。

## スキル早見表（`/<name>` で起動）

| スキル | 用途 |
|---|---|
| `/project-init` | 新規プロジェクトの初期化。**テンプレートから作ったら最初にこれを実行する。** |
| `/qmd-setup` | 検索エンジン qmd のインストール・セットアップ（初回のみ） |
| `/wiki-ingest` | `raw/` のソース（pptx/pdf/xlsx/docx は抽出）をWikiへ取り込み、波及更新 |
| `/wiki-query` | qmdでWikiを検索し、引用付きで回答・還元 |
| `/wiki-lint` | 矛盾・陳腐化・孤立ページ等の健全性チェック |

## 3レイヤー（詳細は `wiki/_schema.md`）

- `raw/` … 生ソース（**不変**。読むだけ） — 真実の出所
- `wiki/` … AIが生成・維持する知識ベース
- `wiki/_schema.md` … Wikiの仕様（スキーマ）

> **可変状態の置き場所**: 「現在のリリースサイクル」など変化する状態は `wiki/overview.md` に一元管理する。このファイルには書かない。
