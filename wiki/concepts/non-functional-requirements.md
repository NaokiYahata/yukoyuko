# 非機能要件（MVP）

## 定義 / 論点
MVPフェーズで満たすべき「最低限」の非機能要件。各社見積もり比較の共通条件。

## 現在の理解
IPA「非機能要求グレード」の6大項目（A〜F）に沿って定義。過剰スペック（24/7監視等）は意図的に除外（[[20260527-non-functional-requirements]]）。

- **A. 可用性**: 目標稼働率99%（月7.2h以内のダウン許容、SLAではなく運用目標）。
- **B. 性能・拡張性**: API応答 p95<0.5秒 / 画面表示 p95<2秒 / アプリ起動コールドスタート3秒以内 / 想定DAU 初期5,000→1年後50,000。
- **C. 運用・保守性**: 監視はクラウド標準アラートのみ / サポート平日9-18時+緊急時ベストエフォート / バックアップ日次・1ヶ月保管 / クラッシュは重大度別管理（重大障害はクラッシュフリー率99.0%以上・24h以内パッチ）。
- **D. 移行性**: 移行データなし（新規構築）。旅館マスタは別途整備。
- **E. セキュリティ**: TLS1.2以上必須 / DB暗号化+パスワードハッシュ化 / 認証はメール・パスワードのBasic（SNS連携は任意、Release 1次第）/ アクセスログ3ヶ月以上 / 通報・ブロック必須。
- **F. システム環境**: iOS 18/26・Android 12〜16対応 / 実機保証は計3端末 / 日本語のみ（多言語化はRelease 2以降）/ 4G LTE以上+Wi-Fi前提。
