# 主要機能・MVPスコープ候補

## 定義 / 論点
アプリが備える機能と、年内リリースMVPに何を含めるか。

## 現在の理解
- MUST機能（現段階で確定）: **宿を予約する機能** と **クチコミを投稿する機能**（[[20260527-app-questions]] Q21）。
- コンセプト上の主要機能: 温泉情報の提供 + ユーザー間コミュニティ機能（意見交換・募集等）（[[20260612-yukoyuko-app-plan]]）。
- 対象施設: **温泉旅館のみ**（その他宿泊施設は対象外）。
- 検索体験: 発注側は要件を絞り込まず、**企画含めての提案**を求める。Web絞り込み（泉質等）の単純再現だけではアプリとして出さない＝アプリ独自の検索・提案体験（例: 移動・行動シナリオベース）を提案余地として残す（Q15）。
- コミュニティ運営は社内で監視・削除・修正（[[content-moderation-and-operation]]）。
- 「音声ルーム」機能が質問に登場するが、発注側は「どの機能で使われるのか伺いたい」と未理解 → リアルタイム通信機能はこちらの提案次第（Q25）。
- 施設/運営側の管理画面は**不要**（Q20）。

## 未解決の問い・矛盾
- MVPに含める機能セットの確定（宿予約・クチコミ以外のコミュニティ機能をどこまで初回に入れるか）。
- 検索・提案体験の具体（絞り込み再現 vs 行動シナリオ提案）はこちらの企画提案で詰める。
- 音声ルーム等リアルタイム機能の要否・採否。
