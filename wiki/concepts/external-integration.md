# 外部システム連携（顧客管理システム）

## 定義 / 論点
アプリが連携する顧客管理システムの素性と連携方式、連携を詰める順序。

## 現在の理解
- 顧客管理システムは **[[yukoyuko]] の自社開発**（パッケージ/SaaSではない）（[[20260527-app-questions]] Q22）。
- 連携方式: API または CSV連携が想定され、マニュアル提示は可能（Q7）。
- **順序の方針**: まず**アプリのUIUX提案を先行**し、その先で「機能を実現するためのAPI等の連携をどうするか」を提示する（またはこちら=開発側で開発する）。API情報の提供はUIUX企画提案の後のやり取り（Q7/Q22）。
- 連携対象データ: 会員アカウント・予約履歴等（会員共通化・双方向同期に必要、[[app-web-role-split]]）。
- 宿情報（旅館マスタ）は新規構築側で別途整備（移行データなし、[[non-functional-requirements]] D-01）。

## 未解決の問い・矛盾
- 顧客管理システムの名称・API仕様・提供範囲は現時点で**意図的に後回し**（UIUX提案後）。
- ログイン情報が顧客管理システム側で管理されているかの確認（Q52は「他質問で回答済み」とされ明示回答が辿れない）。
